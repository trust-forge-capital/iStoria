# iStoria QA 验收清单 v0.1

## 验收目标
确保 iStoria v0.1 在 SSH 场景下可用、输出稳定、跨平台行为可接受，并满足产品定义的核心体验。

## 一、安装与启动

### 1. 单文件运行
- [ ] macOS 可直接运行
- [ ] Linux 可直接运行
- [ ] Windows 可直接运行
- [ ] 二进制名称、权限、执行方式清晰

### 2. 基础可执行性
- [ ] `istoria --help` 正常输出
- [ ] `istoria version` 正常输出
- [ ] 非法命令有清晰报错
- [ ] 非法参数有清晰报错

## 二、核心命令验收

### 3. `istoria stat`
- [ ] 可输出整机摘要
- [ ] 输出包含 host / platform / uptime
- [ ] 输出包含 CPU / Memory / Disk / Network / Sensor / Power 摘要
- [ ] 默认输出在 SSH 终端下易读

### 4. `istoria cpu`
- [ ] 可输出 CPU 总占用
- [ ] 可输出核心数
- [ ] 可输出 load average（平台支持时）
- [ ] Apple Silicon 平台可验证增强字段或增强展示入口

### 5. `istoria mem`
- [ ] 可输出 total / used / available
- [ ] 可输出 used percent
- [ ] swap 字段行为合理

### 6. `istoria disk`
- [ ] 可输出根分区信息
- [ ] used/free/used percent 数值合理
- [ ] 文件系统字段行为合理

### 7. `istoria sensor`
- [ ] 支持平台上可输出温度信息
- [ ] 支持平台上可输出风扇/功耗信息（如可获取）
- [ ] 不支持平台时行为可接受，不直接崩溃

### 8. `istoria net`
- [ ] 可输出主接口
- [ ] 可输出 RX/TX 速率或核心网络信息
- [ ] IP 输出合理

### 9. `istoria power`
- [ ] 在支持设备上可输出电池/电源状态
- [ ] 在不支持设备上有清晰降级行为

## 三、JSON 输出验收

### 10. `--json`
- [ ] `stat --json` 可输出合法 JSON
- [ ] `cpu --json` 可输出合法 JSON
- [ ] `mem --json` 可输出合法 JSON
- [ ] `disk --json` 可输出合法 JSON
- [ ] `sensor --json` 可输出合法 JSON

### 11. JSON 结构一致性
- [ ] 顶层字段与 `json-schema-v0.1.md` 一致
- [ ] `timestamp` 格式正确
- [ ] `platform.os` / `platform.arch` 正确
- [ ] 核心字段命名使用 snake_case
- [ ] 缺失字段处理符合约定（省略优先，不乱填）

## 四、Live 模式验收

### 12. `istoria net --live`
- [ ] 默认 1 秒刷新
- [ ] 数值可持续变化
- [ ] 顶部状态行显示正常
- [ ] `Ctrl+C` 可优雅退出

### 13. `istoria stat --live`
- [ ] 可按固定间隔刷新
- [ ] 输出布局稳定
- [ ] 不因单个字段暂时不可获取而整体崩溃

### 14. `--interval`
- [ ] 可正确设置刷新频率
- [ ] 小于最小值时有合理处理
- [ ] 极端值不会导致明显异常

### 15. `--no-clear`
- [ ] 开启后不清屏
- [ ] 输出可持续追加
- [ ] 适合调试与日志采样

### 16. `--live --json --no-clear`
- [ ] 输出为 JSON Lines
- [ ] 每行均为可解析 JSON
- [ ] 可被 shell / jq / Python 持续消费

## 五、异常与降级行为

### 17. 不支持字段时的表现
- [ ] 不支持传感器的平台不会崩溃
- [ ] 不支持电池的平台不会崩溃
- [ ] 无法识别的增强字段不会污染基础输出

### 18. 单次采集失败
- [ ] 单次采集错误时有清晰提示
- [ ] live 模式不会立即崩溃退出
- [ ] 非 live 模式错误提示可理解

## 六、跨平台一致性

### 19. 一致性检查
- [ ] macOS / Linux / Windows 命令入口一致
- [ ] 核心命令命名一致
- [ ] JSON 基础字段结构一致
- [ ] 平台差异仅体现在扩展字段或缺省字段

## 七、体验验收

### 20. SSH 场景可用性
- [ ] 常规 SSH 终端中可正常显示
- [ ] 一屏内能看懂主要状态
- [ ] 默认输出不过度冗长
- [ ] 用户无需组合多个系统命令即可得到主要结论

## 八、发布前通过标准
- [ ] 核心命令全部通过
- [ ] JSON 输出通过
- [ ] 至少 `net --live` 通过
- [ ] `stat --live` 至少达到可演示/可试用质量，或完成明确降级说明
- [ ] Apple Silicon 路径至少有一台真实设备验证

## 产品验收口径
只要满足以下 5 项，即可认为 v0.1 达到产品最小可用标准：
1. [ ] 用户能下载并运行单文件版本
2. [ ] 用户能通过 `stat/cpu/mem/disk/sensor` 完成基础状态查看
3. [ ] 用户能通过 `--json` 接入脚本
4. [ ] 用户能通过 `net --live` 持续观察当前状态
5. [ ] Apple Silicon 路径存在明确增强能力或增强接口
